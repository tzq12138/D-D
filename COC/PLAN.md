# COC AI Keeper 网页端第一版计划

## Summary
做一个**本机演示版网页应用**：AI 作为守秘人/KP 推故事，软件负责多人房间、角色状态、资料检索、掷骰与COC核心判定。第一版用浏览器多窗口模拟多人，资料既支持扫描本地资料文件夹，也支持网页上传。

技术默认：TypeScript + React/Vite + Express + Socket.IO + SQLite。DeepSeek 用 OpenAI 兼容接口接入，`DEEPSEEK_API_KEY` 放在本地环境变量里；模型默认 `deepseek-v4-pro` 负责叙事，轻量任务可用 `deepseek-v4-flash`。DeepSeek接口依据官方文档：[`base_url=https://api.deepseek.com`](https://api-docs.deepseek.com/)；旧 `deepseek-chat` 将在 2026-07-24 停用，避免首版使用旧模型名。

## Key Changes
- **核心流程**
  - 创建本地房间，玩家输入昵称加入。
  - 主频道公开显示玩家行动、AI叙事、掷骰结果、状态变化。
  - 暂不做复杂私密视角；数据模型保留 `visibility` 字段，后续补私密线索。
  - 人类控制台保留最终控制权：可编辑AI回复、修正状态、手动触发检定、查看日志。

- **AI KP**
  - 玩家发行动后，后端检索已导入资料与当前房间状态，组装提示词调用 DeepSeek。
  - AI 输出结构化结果：叙事文本、是否需要检定、建议技能/难度、可能的状态变化。
  - 需要检定时不直接“口胡结果”，由软件生成检定卡，玩家或控制台点击掷骰，再把结果回喂AI生成后续叙事。

- **规则与角色**
  - 实现 d100、奖励骰、惩罚骰、常规/困难/极难成功、大成功/大失败。
  - 实现技能检定、对抗检定、组合检定的基础逻辑。
  - 实现 SAN 检定表达式，如 `0/1D6`、`1/1D10`；单次损失 >=5 时触发 INT 检定和疯狂发作表。
  - 角色卡包含属性、HP、SAN、幸运、MP、移动、技能、物品、伤口、恐惧/躁狂、成长标记。
  - 战斗、追逐、完整装备价格、地图棋子先记录为二期。

- **资料导入**
  - 启动时扫描本地 `sources/` 文件夹，支持 `.md`、`.txt`、`.pdf`，可扩展 `.xlsx` 角色卡读取。
  - 浏览器支持上传资料文件，上传后进入同一资料库。
  - 资料按段落切片入库，用本地检索给AI提供上下文；第一版先用关键词/全文检索，不上复杂向量库。
  - 规则书和导入内容只作为本机个人资料使用，不把抽取后的版权文本写进公开源码。

## Interfaces
- **主要页面**
  - `/`：创建/加入房间。
  - `/room/:id`：玩家桌面，含聊天、角色卡、检定卡、日志。
  - `/keeper/:id`：控制台，含AI草稿、状态编辑、资料检索结果、手动检定。
  - `/library`：资料扫描状态与网页上传。

- **核心接口**
  - `POST /api/rooms`：创建房间。
  - `POST /api/rooms/:id/join`：加入房间。
  - `POST /api/messages`：提交玩家行动或控制台指令。
  - `POST /api/ai/respond`：生成AI KP回应。
  - `POST /api/rolls`：执行规则检定并广播结果。
  - `POST /api/library/ingest`：扫描本地资料或处理上传文件。

- **核心类型**
  - `Room`：房间、当前场景、时间线、参与者。
  - `Investigator`：角色卡与状态。
  - `RollRequest` / `RollResult`：技能、难度、奖励/惩罚骰、结果等级。
  - `SanEvent`：SAN表达式、损失、临时疯狂检查、发作结果。
  - `KnowledgeChunk`：资料来源、页码/文件、文本片段、检索标签。
  - `KeeperResponse`：AI叙事、检定请求、状态建议、控制台备注。

## Test Plan
- 规则单元测试：d100边界、奖励/惩罚骰取值、成功等级、对抗检定、SAN表达式、疯狂触发。
- 资料测试：导入现有报告和PDF，确认能切片、检索，并在AI提示词里引用来源摘要。
- 多人同步测试：两个玩家窗口和一个控制台窗口同时进入房间，消息、掷骰、状态变化实时同步。
- AI流程测试：有 DeepSeek key 时真实调用；无 key 时用模拟AI返回固定结构，保证演示不阻塞。
- 验收场景：玩家声明“调查图书馆暗门”，AI请求侦查/图书馆使用检定，软件掷骰，AI根据成功/失败推进故事并记录日志。

## Assumptions
- 第一版目标是“跑通最小可玩闭环”，不是完整商业平台。
- 联机先做本机多窗口演示，不做公网账号、权限、部署和安全审计。
- 信息边界先简单公开，后续补私密线索和角色视角。
- AI可当主要KP，但人类控制台必须存在，用来兜底和修正。
- 二期再做完整战斗、追逐、私密线索、AI一致性审查、素材生成和公网部署。
