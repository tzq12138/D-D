# 📖 AI-Keeper 前端架构与人机交互(HCI)设计白皮书 (v1.0)

## v1.1 修订说明

前端交互必须遵守 `HOST_PROTOCOL_CONTRACT.md`：

- Host 端是公共演出终端，只消费 Engine 事件，不发起业务写操作。
- Player 端快捷按钮只能提交玩家意图，例如 `submit_player_action`，不能直接触发状态修改。
- 掷骰、伤害、SAN、线索解锁等公共信息必须通过 `s2c_reveal_transaction` 控制展示顺序。
- 氛围变化由 Engine 广播 `s2c_atmosphere`；AI 只能提交氛围建议，不能直接操纵 Host。

## 一、 宏观前端形态：非对称客户端架构 (Asymmetric Client Architecture)
为了彻底解决线下跑团（面团）的抢话问题并实现极致的沉浸感，前端采用“一主多从”的非对称渲染模式。

1. **Host 端（大屏幕/公共视听中心）**
   * **定位**：桌子中央的公共显示屏，作为 AI KP 的化身。
   * **功能**：播放全局 BGM、展示场景动态背景图、渲染 3D 掷骰动画、流式打印 AI 剧情文本、显示全体调查员的简易血条/理智槽。无输入框。
2. **Player 端（手机/私人战术手台）**
   * **定位**：玩家个人的交互控制器与私密情报板。
   * **功能**：Push-to-Talk（按住说话）语音输入、私密状态更新震动、完整的角色卡与背包管理、接收系统单播（Unicast）的秘密线索。

---

## 二、 渲染队列与防剧透控制 (The Render Queue & UI Lock)
为防止 WebSocket 的毫秒级响应（如大失败指令）比 3D 骰子动画更早上屏从而导致剧透，前端必须实现基于状态锁的异步队列。

* **数据结构**：分离“底层通信缓冲区（`eventQueue`）”与“视图展示区”。
* **核心机制 (The UI Lock)**：
  当遇到 `s2c_reveal_transaction` 时，前端使用 `lockToken` 上锁。
  此时后续到达的 AI 文本流和状态变化将被阻塞在队列中。待 3D 骰子、状态变化、转场等步骤按事务顺序完成后，组件调用 `releaseLock(lockToken)` 解锁，积压的绝望/兴奋文本如潮水般涌出。

---

## 三、 系统延迟的拟真化包装 (State Affordance)
不使用传统的转圈 Loading，而是将大模型与 RAG 检索的延迟包装成游戏悬念。

* **状态侦听**：前端根据后端的 `s2c_engine_state` 指令切换状态指示器。
  - `retrieving_memory` -> *“👁️ 守秘人正在翻阅旧日笔记...”*
  - `rolling_hidden_dice` -> *“🎲 守秘人正在帷幕后进行一次暗骰...”*
* **压迫感升级机制 (Escalation)**：当卡顿或等待超过 3 秒，文案自动变更为 *“时间一分一秒地流逝，某种不可名状的直觉让你感到不安...”*，并伴随 UI 的轻微呼吸或失真特效，将网络延迟转化为心理压迫感。

---

## 四、 动态输入权限与富文本动作 (Smart Input & Inline Actions)
把聊天框升级为智能战术终端，解放玩家的双手。

1. **富文本按钮注入**：AI 在生成的文本中使用 `[动作|意图|参数]` 格式。前端将其渲染为可点击按钮（如 `[🎲 侦查]`）。玩家点击只提交玩家动作意图，由 Engine 判定是否需要检定、是否落库状态变化。
2. **输入框状态锁**：在系统结算或播放动画时，输入框变灰并提示 *“⏳ 等待守秘人结算中...”*，防止玩家乱发指令导致状态机错乱。
3. **私人快捷指令盘**：根据玩家角色卡动态生成的常驻按钮（如“开枪”、“话术”），战斗状态下自动切换为战术指令。

---

## 五、 多感官氛围动态调度 (Immersive Atmosphere Engine)
在 React 顶层挂载 `<AtmosphereOverlay />` 覆盖层和全局 `AudioManager`。

* **协议支持**：通过 `s2c_atmosphere` 协议接收环境指令。
* **视觉特效**：支持动态暗角（Vignette）、血红滤镜、屏幕剧烈震动（Camera Shake）、故障雪花屏（Glitch）。
* **听觉特效**：使用 Howler.js 实现 BGM 的无缝淡入淡出（Cross-fade），并在大失败或掉血时叠加急促的心跳或骨折音效。
* **触发机制**：由 Python 引擎在检测到状态剧变时触发；AI 只能调用氛围建议工具，最终是否广播由 Engine 决定。

---

## 六、 动态情报板与线索木板 (Dynamic HUD & Clue Board)
彻底解放玩家的记忆负担，系统自动记账。

1. **活体角色卡**：理智值碎裂动画、濒死红框闪烁。点击面板上的技能数值直接向系统发起暗骰。
2. **互动式背包**：获得物品时弹出通知。背包内物品可点击展开菜单（检视/使用/展示给队友）。
3. **调查员软木板**：后端 `unlocked_knowledge` 的可视化。自动把解锁的人物、地点、物品以“宝丽来照片+图钉+红线”的形式钉在 UI 木板上，随时查阅，消除信息差恐惧。
